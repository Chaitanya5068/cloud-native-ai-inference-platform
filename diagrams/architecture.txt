DATA FLOW:

1. User/Client Request
   └─> HTTP POST /infer to Public IP:80
       └─> API Gateway (FastAPI)
           └─> Internal VPC request to Worker:8001
               └─> Python Worker receives request
                   └─> Forwards to TypeScript Worker:8002
                       └─> TypeScript Worker validates
                           └─> Forwards to Model Worker:8003
                               └─> Model Worker generates response
                                   └─> Returns response back through pipeline
                                       └─> API Gateway returns JSON to client

2. Health Checks
   └─> All services expose /health endpoint
   └─> Monitored by Docker health checks
   └─> Can be checked via curl or monitoring tools

SECURITY FEATURES:

1. Network Isolation
   - Public and private subnets
   - Worker server has no public IP
   - NAT Gateway prevents direct internet access to workers

2. Security Groups
   - Principle of least privilege
   - SSH access restricted to your IP
   - Worker communication only within VPC

3. No Data Exposure
   - Private subnet communication is internal
   - No sensitive data exposed to internet

PRODUCTION IMPROVEMENTS:

1. HTTPS/TLS
   - Use AWS Certificate Manager (ACM)
   - Configure HTTPS on API Gateway
   - Encrypt worker communication

2. Load Balancing
   - Add Application Load Balancer (ALB)
   - Distribute traffic across multiple API instances

3. Auto Scaling
   - Configure Auto Scaling Groups
   - Scale based on CPU/Memory utilization

4. Monitoring & Logging
   - CloudWatch for logs
   - Prometheus/Grafana for metrics
   - Health checks and alerts

5. Container Orchestration
   - Migrate to Kubernetes (EKS)
   - Better scalability and management
   - Rolling deployments

6. Data Persistence
   - RDS for database
   - ElastiCache for caching
   - S3 for object storage

7. Secrets Management
   - AWS Secrets Manager
   - Rotate credentials regularly
   - Never commit secrets to Git
